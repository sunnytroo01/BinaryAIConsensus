HTML Forms Deep Dive. Everything about collecting user input on the web.

A form is the bridge between the user and the server. When a user fills out a form and clicks submit, the browser packages the data and sends it to the server. The server processes the data and responds. Every login page, every checkout page, every search bar, every contact form, every signup flow uses HTML forms.

Form attributes that matter:

action is the URL where the form data is sent. action="/api/submit" sends data to your server. action="https://formspree.io/f/xyz" sends to a third-party form handler.

method is either GET or POST. GET appends data to the URL as query parameters, visible in the browser address bar. Use GET for search forms and filters. POST sends data in the request body, not visible in the URL. Use POST for login forms, signups, and any form that sends sensitive data.

enctype matters for file uploads. Use enctype="multipart/form-data" when the form includes file inputs.

Input validation with HTML attributes:

required makes a field mandatory. The form will not submit until this field is filled.
<input type="text" required>

minlength and maxlength set character limits.
<input type="text" minlength="3" maxlength="50">

min and max set numeric ranges.
<input type="number" min="1" max="100">

pattern sets a regex pattern for validation.
<input type="text" pattern="[A-Za-z]{3,}" title="At least 3 letters">

The placeholder attribute shows hint text inside the field.
<input type="email" placeholder="you@example.com">

Labels are essential for accessibility. Screen readers read the label text to describe the input. Always associate labels with inputs using the for attribute.

<label for="username">Username</label>
<input type="text" id="username" name="username">

Fieldset and legend group related inputs visually and semantically.

<fieldset>
<legend>Shipping Address</legend>
<label for="street">Street</label>
<input type="text" id="street" name="street">
<label for="city">City</label>
<input type="text" id="city" name="city">
<label for="zip">ZIP Code</label>
<input type="text" id="zip" name="zip" pattern="[0-9]{5}" title="5-digit ZIP code">
</fieldset>

Datalist provides autocomplete suggestions.

<input type="text" list="colors" placeholder="Pick a color">
<datalist id="colors">
<option value="Red">
<option value="Blue">
<option value="Green">
<option value="Yellow">
</datalist>

Output element displays calculation results.

<form oninput="result.value = parseInt(a.value) + parseInt(b.value)">
<input type="range" id="a" value="50"> +
<input type="number" id="b" value="25"> =
<output name="result" for="a b">75</output>
</form>

Button types: submit sends the form. reset clears all fields. button does nothing by default but can be wired to JavaScript.

<button type="submit">Send</button>
<button type="reset">Clear</button>
<button type="button" onclick="doSomething()">Click Me</button>

Complete signup form example:

<form action="/api/signup" method="POST">
<h2>Create Account</h2>

<label for="fullname">Full Name</label>
<input type="text" id="fullname" name="fullname" required minlength="2" maxlength="100" placeholder="Jane Smith">

<label for="email">Email Address</label>
<input type="email" id="email" name="email" required placeholder="jane@example.com">

<label for="password">Password</label>
<input type="password" id="password" name="password" required minlength="8" placeholder="At least 8 characters">

<label for="dob">Date of Birth</label>
<input type="date" id="dob" name="dob" required>

<label for="country">Country</label>
<select id="country" name="country" required>
<option value="">Select your country</option>
<option value="us">United States</option>
<option value="uk">United Kingdom</option>
<option value="ca">Canada</option>
<option value="au">Australia</option>
</select>

<div>
<input type="checkbox" id="terms" name="terms" required>
<label for="terms">I agree to the Terms of Service</label>
</div>

<button type="submit">Create Account</button>
</form>

Every input should have a name attribute. The name is what the server uses to identify the data. Without a name, the input data is not sent with the form submission. The id is used for label association and JavaScript targeting. The name is used for data submission. They can be the same value but serve different purposes.
