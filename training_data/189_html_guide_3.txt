HTML Accessibility and SEO. Making websites usable by everyone and findable by search engines.

Accessibility means making websites usable by people with disabilities including visual, auditory, motor, and cognitive impairments. Approximately 15% of the world population has some form of disability. An accessible website is not just morally right. It is legally required in many jurisdictions and it improves the experience for all users.

Alt text on images is the most basic accessibility requirement. Every img element should have an alt attribute that describes the image content. Screen readers read this text aloud to visually impaired users.

Good alt text: <img src="chart.png" alt="Bar chart showing revenue growth from $1M in 2020 to $4M in 2024">
Bad alt text: <img src="chart.png" alt="chart"> or <img src="chart.png" alt="">

For decorative images that convey no information, use an empty alt attribute: <img src="decorative-line.png" alt=""> This tells screen readers to skip the image.

Semantic HTML is the foundation of accessibility. Use headings h1 through h6 in order. Use nav for navigation. Use main for primary content. Use button for clickable actions. Use a for navigation links. Screen readers use these semantic elements to build a structural overview of the page that users can navigate quickly.

ARIA attributes add accessibility information when HTML semantics are not sufficient. aria-label provides a text label for elements without visible text. aria-describedby links an element to a description. aria-hidden="true" hides decorative elements from screen readers. role defines the purpose of an element.

<button aria-label="Close dialog">X</button>
<div role="alert">Your form has been submitted successfully.</div>
<input type="text" aria-describedby="help-text">
<p id="help-text">Enter your email address. We will never share it.</p>

Keyboard navigation is essential. All interactive elements must be reachable and operable using only the keyboard. Use tabindex="0" to make custom elements focusable. Never remove the focus outline without providing an alternative. Users who cannot use a mouse rely entirely on keyboard navigation.

Color contrast must meet WCAG guidelines. Text must have a contrast ratio of at least 4.5:1 against its background for normal text and 3:1 for large text. Do not convey information through color alone since colorblind users may not distinguish between colors.

Skip links allow keyboard users to skip repeated navigation and jump directly to the main content. Place a hidden link at the top of the page.

<a href="#main-content" class="skip-link">Skip to main content</a>
<nav>...</nav>
<main id="main-content">...</main>

SEO means Search Engine Optimization. It makes your pages rank higher in Google and other search engines.

The title tag is the most important SEO element. It appears in browser tabs and search results. Keep it under 60 characters. Include your primary keyword. Make it compelling.

<title>How to Train a Dog - Complete Guide for Beginners | PetCare</title>

Meta description appears below the title in search results. Keep it between 150-160 characters. Include your primary keyword. Make it a compelling summary that encourages clicks.

<meta name="description" content="Learn how to train your dog with our step-by-step guide. Covers basic commands, house training, leash training, and behavior correction for all breeds.">

Heading structure matters for SEO. Use one h1 per page containing your primary keyword. Use h2s for main sections. Use h3s for subsections. Search engines use heading structure to understand the content hierarchy.

URL structure should be clean and descriptive. Good: /blog/how-to-train-a-dog. Bad: /blog/post?id=12345. Include keywords in URLs. Use hyphens to separate words.

Image optimization for SEO: Use descriptive file names (golden-retriever-playing.jpg not IMG_4521.jpg). Add alt text with relevant keywords. Compress images to reduce file size and improve page load speed. Use modern formats like WebP.

Schema markup provides structured data that search engines can understand. It enables rich snippets in search results like star ratings, prices, and event dates.

<script type="application/ld+json">
{
  "@context": "https://schema.org",
  "@type": "Article",
  "headline": "How to Train a Dog",
  "author": {"@type": "Person", "name": "Jane Smith"},
  "datePublished": "2025-03-15",
  "description": "Complete guide to dog training for beginners."
}
</script>

Page speed is a ranking factor. Minimize CSS and JavaScript file sizes. Compress images. Use lazy loading for images below the fold. Minimize HTTP requests. Use a CDN for static assets.

<img src="photo.jpg" alt="Description" loading="lazy">

Mobile-friendliness is essential for SEO since Google uses mobile-first indexing. Always include the viewport meta tag and use responsive design.
