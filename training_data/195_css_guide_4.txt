CSS Responsive Design and Media Queries. Making websites work on every screen size.

Responsive design means building websites that adapt their layout and styling to the screen they are viewed on. A responsive website looks great on a phone, a tablet, a laptop, and a large desktop monitor without needing separate versions for each device.

The viewport meta tag is required for responsive design:
<meta name="viewport" content="width=device-width, initial-scale=1.0">
Without this tag, mobile browsers render the page at a desktop width and zoom out.

Mobile-first approach means writing your base CSS for small screens and then adding styles for larger screens with media queries. This is better than desktop-first because mobile CSS is simpler and smaller, resulting in faster load times on mobile devices.

Media queries apply CSS rules only when conditions are met:

@media (min-width: 768px) {
  .container { max-width: 720px; }
}

@media (min-width: 1024px) {
  .container { max-width: 960px; }
}

@media (min-width: 1200px) {
  .container { max-width: 1140px; }
}

Common breakpoints: 480px for large phones, 768px for tablets, 1024px for laptops, 1200px for desktops, 1400px for large desktops. These are guidelines, not rules. Set breakpoints where your design actually breaks, not at arbitrary device widths.

Responsive typography with clamp:
font-size: clamp(16px, 2.5vw, 24px);
This sets a minimum of 16px, a preferred size that scales with viewport width, and a maximum of 24px. The font smoothly scales between the min and max.

Responsive images:
img { max-width: 100%; height: auto; }
This ensures images never exceed their container width and maintain aspect ratio.

For art direction, use the picture element:
<picture>
<source media="(min-width: 1024px)" srcset="large.jpg">
<source media="(min-width: 768px)" srcset="medium.jpg">
<img src="small.jpg" alt="Description">
</picture>

Responsive navigation pattern. On mobile, show a hamburger menu. On desktop, show horizontal links.

.nav-links { display: none; }
.hamburger { display: block; }

@media (min-width: 768px) {
  .nav-links { display: flex; gap: 20px; }
  .hamburger { display: none; }
}

CSS custom properties (variables) for responsive design:

:root {
  --spacing: 16px;
  --font-size: 16px;
  --max-width: 100%;
}

@media (min-width: 768px) {
  :root {
    --spacing: 24px;
    --font-size: 18px;
    --max-width: 720px;
  }
}

@media (min-width: 1024px) {
  :root {
    --spacing: 32px;
    --max-width: 960px;
  }
}

.container { max-width: var(--max-width); padding: var(--spacing); margin: 0 auto; }
body { font-size: var(--font-size); }

CSS transitions add smooth animations to property changes:
.button { background: #3b82f6; transition: background 0.2s ease, transform 0.1s ease; }
.button:hover { background: #2563eb; }
.button:active { transform: scale(0.97); }

CSS animations for more complex motion:
@keyframes fadeIn {
  from { opacity: 0; transform: translateY(20px); }
  to { opacity: 1; transform: translateY(0); }
}
.card { animation: fadeIn 0.4s ease forwards; }

Dark mode with prefers-color-scheme:
:root { --bg: #ffffff; --text: #111111; }
@media (prefers-color-scheme: dark) {
  :root { --bg: #0a0a0f; --text: #e4e4e7; }
}
body { background: var(--bg); color: var(--text); }

This automatically switches between light and dark themes based on the user's operating system preference. No JavaScript needed.
